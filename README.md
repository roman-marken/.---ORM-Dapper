# Практичне завдання: список розсилки та акції (Dapper)


## Модель бази (завдання 1)

- **Countries** — країна (для міст і для акцій).
- **Cities** — місто, належить країні.
- **Sections** — тематичні розділи (наприклад, мобільні телефони, ноутбуки).
- **Buyers** — ПІБ (`FullName`), дата народження (`BirthDate`), стать (`Gender`), email (`Email`), місто (`CityId`; країну визначає через місто).
- **BuyerSections** — у яких розділах зацікавлений покупець (зв’язок «багато до багатьох»).
- **Promotions** — акційний товар: назва (`ProductTitle`), розділ (`SectionId`), **країна акції** (`CountryId`), дата початку й кінця (`StartDate`, `EndDate`).

Схему створює програма автоматично при першому успішному підключенні до SQL Server LocalDB (база `MailingListPromoUkDb`).

## Завдання 2 — підключення / від’єднання

Після вибору **1** виконується створення БД/таблиць (за потреби) і відкривається з’єднання. Виводиться повідомлення про успіх із даними сервера, назви бази, версії та стану. Якщо з’єднатися не вдалося — виводиться текст помилки. Пункт **2** закриває з’єднання.

Інтерфейс консолі — **українською**.



Потрібні: .NET SDK 8, SQL Server Express LocalDB або змініть рядок підключення в `Program.cs`.
